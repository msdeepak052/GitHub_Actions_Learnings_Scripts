## âœ… Context

In GitHub Actions, you can:

* Use **services**: temporary containers like databases, caches, etc.

## ðŸ”¹ 1. **Shared Network Using `services:` in GitHub Actions**

GitHub automatically **creates a Docker network** and attaches the main container and all services to it.

### âœ… Example: App Container Talks to MySQL Service

```yaml

  test:
    runs-on: ubuntu-latest
    container:
      image: node:18
    services:
      mysql:
        image: mysql:5.7
        env:
          MYSQL_ROOT_PASSWORD: password
          MYSQL_DATABASE: testdb
        ports:
          - 3306:3306
        options: >-
          --health-cmd="mysqladmin ping -h 127.0.0.1 -p$MYSQL_ROOT_PASSWORD"
          --health-interval=10s
          --health-timeout=5s
          --health-retries=3

      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Install mysql client
        run: apt-get update && apt-get install -y default-mysql-client
```

> ðŸ”¸ `mysql` here is the hostname â€” resolved via the shared Docker network GitHub Actions sets up.

## ðŸ”¹ 2. **Shared Volume Between Containers (Using Docker Compose)**

GitHub Actions doesn't directly support volumes between services or across jobs, but you can use **`docker-compose`** to create your own network and volume setup.

### âœ… Example: Shared Volume Between `app` and `logger`

ðŸ“ `.github/workflows/docker-compose-volume.yml`

```yaml
name: Shared Volume with Compose

on: [push]

jobs:
  compose:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Set up Docker Compose
        run: |
          docker-compose -f docker-compose.yml up -d
          sleep 10
          docker-compose logs

      - name: Teardown
        run: docker-compose -f docker-compose.yml down
```

ðŸ“ `docker-compose.yml`

```yaml

services:
  app:
    image: busybox
    volumes:
      - shared-data:/data
    command: sh -c "echo Hello > /data/message.txt && sleep 20"

  logger:
    image: busybox
    volumes:
      - shared-data:/data
    command: sh -c "sleep 5 && cat /data/message.txt && sleep 10"

volumes:
  shared-data:
```
